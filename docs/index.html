🔒

本地优先，隐私优先

所有订阅、文章、收藏都存在浏览器本地 IndexedDB，没有任何云端同步。全文抓取后离线也能读，数据始终只属于你。

⚙

自研轻量解析器

不引入第三方 RSS 解析库，自研 XML 标记化解析器，处理 CDATA、HTML 实体、编码检测与相对链接。更轻、更可控。

▤

三栏高效阅读

订阅列表 / 文章列表 / 阅读器三栏可拖拽调宽，虚拟滚动撑起海量文章。一眼看清信息全貌，专注阅读不打断。

✦

智能信息筛选

过滤规则引擎自动归类与降噪，AI 每日简报按重要性挑出当天最值得看的几条。让信息流变成知识库。

看看它怎么帮你阅读

三栏架构、过滤引擎、AI 简报、离线隐私——逐个拆给你看。

三栏架构

信息全貌与专注阅读，一次给齐

侧边栏（订阅）、中间栏（文章列表）、右侧阅读器同时展开。两道分隔条都可拖拽调宽：侧栏 180–480px、文章列表 280–700px，宽度记忆在本地。文章列表用 react-virtuoso 虚拟滚动，几千篇文章也能丝滑翻阅。

  • 可调栏宽 — 拖拽分隔条，宽度持久化到本地
  • 虚拟滚动 — 仅渲染可见项，海量文章不卡顿
  • 状态实时同步 — 已读 / 未读 / 星标即时更新

过滤规则引擎

把噪音，交给规则去处理

按标题、内容、作者、链接设置条件，支持正则精确匹配，AND / OR 自由组合。命中后可标记已读、加星、删除或打标签。规则可以是「全局」也可以「只对某个订阅源」生效——让招聘、营销、水文自动归位，阅读列表只剩你真正想看的。

  • 四个字段 — title / content / author / url
  • 正则 + 逻辑 — matches 操作符 + AND/OR 组合
  • 四种动作 — 标记已读 · 加星 · 删除 · 加标签
  • 作用域 — 全局规则 或 按订阅源规则

AI 每日简报

每天最重要的几条，AI 帮你挑好

后台定时汇总过去 24 小时的文章，调用你配置的 OpenAI 兼容接口，按重要性分成高 / 中 / 低三档，挑出 5–10 条最值得看的。单篇文章也能一键生成结构化摘要与关键词标签。不配 API 也完全不影响阅读，AI 是锦上添花。

  • 每日简报 — 自动汇总，按高 / 中 / 低重要性分类
  • 文章摘要 — 核心概括 + 关键要点 + 关键词标签
  • OpenAI 兼容 — 自定义端点、Key、模型

隐私与离线

你的订阅，从未离开过你的浏览器

没有账号、没有云端、没有同步服务器。订阅、文章、文件夹、过滤规则、设置，全部写进本地 IndexedDB。配合 Mozilla Readability 全文抓取，断网时也能把已经下载的文章完整读完。

  • 零云端 — 数据只在本地，无任何远程写入
  • 全文抓取 — Readability 提取正文，离线可读
  • OPML 迁移 — 导入 / 导出，订阅一键搬走

安装与使用

当前为开发者加载方式，暂未上架 Chrome 应用商店。

  1. 1

    克隆仓库

    git clone git@github.com:Michaelooo/chrome-rss.git
  2. 2

    安装依赖并构建

    pnpm install && pnpm build
  3. 3

    加载到 Chrome

    打开 chrome://extensions，开启右上角「开发者模式」，点击「加载已解压的扩展程序」，选择项目根目录下的 dist/ 文件夹。

  4. 4

    添加订阅（可选 AI）

    点击侧栏 ➕ 添加 RSS Feed 地址。如需 AI 摘要与每日简报，进入设置页填写 OpenAI 兼容端点、Key 与模型；不填也完全可用。
